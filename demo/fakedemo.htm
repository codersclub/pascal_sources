FILES IN THE PACKAGE
~~~~~~~~~~~~~~~~~~~~

FAKE Demostration is distributed in two ZIP files named:

  FAKE.ZIP and
  FAKESRC.ZIP

You must unpack the first zip-file in your hard disk before
the demo can be run. After unpacking both files, you should
have the following files and directories:

  README.TXT      - This file
  FAKE.EXE        - Demo executable file
  FILE_ID.DIZ     - BBS file descriptions
  SOURCE\*.*      - Demo parts source code



HARDWARE REQUIREMENTS
~~~~~~~~~~~~~~~~~~~~~

This demo requires a 80386 processor and 528 kilobytes of
free conventional memory. You can use a 386 memory manager
like QEMM to get this memory available.

I recommend that you have a 486-based machine, a fast VGA card
and a Sound Blaster or compatible sound card. This will make the
demo more fun to watch. If your machine is 386-based, the demo
will work, but many parts slow down.




STARTING THE DEMO
~~~~~~~~~~~~~~~~~

You can start the demo at your DOS prompt command line executing
the file FAKE.EXE, typing the word FAKE (for example C:\>FAKE).

If the demo hangs or run very slowly or you don't have enough
memory, try the following:

To get memory try installing a 386 memory manager (such as
QEMM 386 or 386MAX) and removing all unnecessary residents
from config.sys and autoexec.bat.

If the demo doesn't work properly but you have enough memory,
first try removing all disk caches and other active resident
programs.
